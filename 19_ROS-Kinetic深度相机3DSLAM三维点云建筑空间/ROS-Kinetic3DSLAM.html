关于ROS的推荐学习资料和流程：

ROS配置

具体参考ROS机器人操作系统。

此次实验选择的ROS版本为"kinetic"，ubuntu版本为“Ubuntu 16.04.4 LTS”，其代号“xenial”。

ROS安装流程

#添加软件源到sources.list文件中
sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'
#设置密码，确保原始代码是正确的。
sudo apt-key adv --keyserver hkp://ha.pool.sks-keyservers.net:80 --recv-key 421C365BD9FF1F717815A3895523BAEEB01FA116
#更新软件包，避免错误的或不对的软件版本带来的潜在问题。
sudo apt-get update
#推荐完全安装。
sudo apt-get install ros-kinetic-desktop-full
#查看可获取的包
apt-cache search ros-kinetic 
#初始化rosdep，确保安装所需依赖。
sudo rosdep init
rosdep update
#自动添加ROS环境变量，保证每新启动shell自动添加到bash会话中。“.”为隐藏文件，可通过“ctrl+h”显示与隐藏。
echo "source /opt/ros/kinetic/setup.bash" >> ~/.bashrc
source ~/.bashrc

安装完成后可以通过执行roscore来查看是否已正确安装。

建立ROS工作区(workspace)

可以建立多个工作区，但是任何时候只能在一个工作区中工作。

mkdir -p ~/catkin_ws/src
cd ~/catkin_ws/
catkin_make

建立的src文件夹放置未来所有的待建立的ROS包(可以包含多个)，例如操控某个机器人的程序包。catkin_make命令首次在workspace中运行时，会建立基本的文件系统结构，如下图：

为了便于在bash中查看文件夹层级结构，安装sudo apt-get install tree，执行tree、tree -L 2查看，或tree -L 2 > ~/tree.txt将路径结构保存到.txt文本中。如下图。


在src文件夹中创建了一个CMakeLists.txt链接。以及build和devel文件夹及其内容。需要注意在devel文件夹中的setup.bash文件，为了避免每次重启bash后，都需要重新输入命令source devel/setup.bash，可以将其写入~/.bashrc中。

echo "source ~/catkin_ws/devel/setup.bash" >> ~/.bashrc
source ~/.bashrc

在src文件加下建立自己的ROS功能包（机器人程序）

cd ~/catkin_ws/src
catkin_create_pkg myrobot std_msgs rospy roscpp
cd ~/catkin_ws
catkin_make

在“myrobot”功能包下，新建几个约定命名的文件夹：“src”（前往通过mkdir -p ~/catkin_ws/src已建立）用于放置.py或者.cpp文件执行文件(亦可放置于新建“scripts”文件夹中)，“msg”放置自定义的消息类型，“srv”用于放置服务文件，“action”用于放置动作(机制)，“launch”放置.launch多节点启动文件。一般是在需要时再建立对应类别的文件夹。

libfreenect2

Linux(Ubuntu 16.04 | intel)

  • Download libfreenect2 source

    git clone https://github.com/OpenKinect/libfreenect2.git
cd libfreenect2

  • Install build tools

    sudo apt-get install build-essential cmake pkg-config
  • Install libusb. The version must be >= 1.0.20.
    sudo apt-get install libusb-1.0-0-dev
  • Install TurboJPEG
    sudo apt-get install libturbojpeg libjpeg-turbo8-dev
  • Install OpenGL
    sudo apt-get install libglfw3-dev
  • Install OpenCL (optional)
    sudo apt-get install beignet-dev
  • Install VAAPI (optional, Intel only)
    sudo apt-get install libva-dev libjpeg-dev
  • Install OpenNI2 (optional)
    sudo apt-get install libopenni2-dev

  • Build (if you have run cd depends previously, cd .. back to the libfreenect2 root directory first.)。判断是否sudo，获取执行权限。

    mkdir build && cd build
cmake .. -DCMAKE_INSTALL_PREFIX=$HOME/freenect2
make
sudo make install

    You need to specify cmake -Dfreenect2_DIR=$HOME/freenect2/lib/cmake/freenect2for CMake based third-party application to find libfreenect2.

    cmake .. -Dfreenect2_DIR=$HOME/freenect2/lib/cmake/freenect2

  • Set up udev rules for device access:sudo cp ../platform/linux/udev/90-kinect2.rules /etc/udev/rules.d/,then replug the Kinect.
    测试

  • Run the test program:./bin/Protonect
  • Run OpenNI2 test (optional):sudo apt-get install openni2-utils && sudo make install-openni2 && NiViewer2,Environment variableLIBFREENECT2_PIPELINEcan be set to cl, cuda, etc to specify the pipeline.
    ./bin/Protonect cpu
./bin/Protonect gl
./bin/Protonect cl

ROS Workspace下IAI Kinect2安装

  1. Install the ROS. Instructions for Ubuntu
  2. Setup your ROS environment
  3. Install libfreenect2:
    Follow the instructions and enable C++11 by using cmake .. -DENABLE_CXX11=ON instead of cmake ..
    If something is not working, check out the latest stable release, for example git checkout v0.2.0.

  4. Clone this repository into your catkin workspace, install the dependencies and build it:

    cd ~/catkin_ws/src/
git clone https://github.com/code-iai/iai_kinect2.git
cd iai_kinect2
rosdep install -r --from-paths .
cd ~/catkin_ws
catkin_make -DCMAKE_BUILD_TYPE="Release"

    catkin_make出错,报错信息如下：
    /opt/ros/kinetic/include/opencv-3.2.0-dev/opencv2/flann/saving.h63: error: exception handling disabled, use -fexceptions to enable throw FLANNException("Invalid index file, cannot read");
    问题出在kinect_registration，在kinect_registration的package里，找到CmakeLists.txt,添加如下 :
    add_definitions( -fexceptions )

  5. Connect your sensor and run kinect2_bridge:

    roslaunch kinect2_bridge kinect2_bridge.launch

    新建一个终端，显示运行图像

    rosrun kinect2_viewer kinect2_viewer

    Add the calibration files to the kinect2_bridge/data/<serialnumber> folder. Further details
    Restart kinect2_bridge and view the results using rosrun kinect2_viewer kinect2_vcdiewer kinect2 sd cloud.

    从图中可以看到，深度数据与图像有偏移，需要对kinect2标定calibration，具体参考地址，主要摘录如下：

Detailed steps:
0.If you haven't already, start the kinect2_bridge with a low number of frames per second (to make it easy on your CPU): rosrun kinect2_bridge kinect2_bridge _fps_limit:=2
1.create a directory for your calibration data files, for example: mkdir ~/kinect_cal_data; cd ~/kinect_cal_data
2.Record images for the color camera: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 record color
3.Calibrate the intrinsics: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 calibrate color
4.Record images for the ir camera: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 record ir
5.Calibrate the intrinsics of the ir camera: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 calibrate ir
6.Record images on both cameras synchronized: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 record sync
7.Calibrate the extrinsics: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 calibrate sync
8.Calibrate the depth measurements: rosrun kinect2_calibration kinect2_calibration chess5x7x0.03 calibrate depth
9.Find out the serial number of your kinect2 by looking at the first lines printed out by the kinect2_bridge. The line looks like this: device serial: 012526541941
·0.Create the calibration results directory in kinect2_bridge/data/$serial: roscd kinect2_bridge/data; mkdir 012526541941
11.Copy the following files from your calibration directory (~/kinect_cal_data) into the directory you just created: calib_color.yaml calib_depth.yaml calib_ir.yaml calib_pose.yaml
12.Restart the kinect2_bridge and be amazed at the better data.

标定之后，数据有所改善。

ROS下安装rtabmap

RTAB-Map (Real-Time Appearance-Based Mapping)是一种基于全局贝叶斯闭环检测的RGB-D Graph SLAM方法。可以用kinect的深度信息结合kinect变换得到的激光数据进行即时定位与建图。
rtabmap的安装参考地址。此次实验是在ROS下使用rtabmap因此选择If ROS is already installed 的安装方法。关于教程可以从官方网站中获取。实验中使用Hand-held Kinect RGB-D Mapping手持kinect 2三维建模的方式。ROS下bash命令为启动rtabmap：

roslaunch kinect2_bridge kinect2_bridge.launch publish_tf:=true

rosrun tf static_transform_publisher 0 0 0 -1.5707963267948966 0 -1.5707963267948966 camera_link kinect2_link 100 

roslaunch rtabmap_ros rtabmap.launch rtabmap_args:="--delete_db_on_start" rgb_topic:=/kinect2/qhd/image_color_rect depth_topic:=/kinect2/qhd/image_depth_rect camera_info_topic:=/kinect2/qhd/camera_info
