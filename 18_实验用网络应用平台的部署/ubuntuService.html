VNC配置

很多时候主机并不需要直接连接显示器，例如作为服务器或者机器人的主机时，但是很多时候希望直接通过常用的电脑(win、maxOS或者Linux系统，甚至ios、Android、Raspberry Pi)来控制，就可以在Ubuntu系统下安装X11VNC Server。

  1. 在Terminal终端安装X11VNC Server
sudo apt-get install x11vnc
  1. 设置VNC密码
x11vnc -storepasswd
  1. 避免每次手工启动X11VNC,则执行以下代码。
sudo x11vnc -storepasswd in /etc/x11vnc.pass
sudo cp ~/.vnc/passwd /etc/x11vnc.pass (被认为为非root的其它管理员)
sudo vim /etc/init/x11vnc.conf （如果没有该文件则创建该文件）

将下述代码，复制到x11vnc.conf文件中。

start on login-session-start
script
x11vnc -display :0 -auth /var/run/lightdm/root/:0 -forever -bg -o /var/log/x11vnc.log -rfbauth /etc/x11vnc.pass -rfbport 5900
end script
  1. 在windows系统下或其它系统，下载VNC Viewer，安装后打开输入预备的电脑在局域网中的地址，以及账户和密码，登录到预备电脑中。(在Ubuntu系统终端中执行ifconfig中查看地址)。

如果希望在iphone中控制，则可以在其手机App Store中下载VNC Viewer-Remote Desktop应用，使用方法同前。

如果不在Ubuntu的终端中安装VNC，则可以在其Ubuntu Software Center中搜索VNC并直接安装。

2. 建立uWSGI入口点(entry points)

建立wsgi.py文件，作为应用程序App的入口点，这将告诉uWSGI服务器如何与应用程序进行交互。

nano wsgi.py

wsgi.py文件代码。从flask项目中主文件lAEVAL.py中调入app应用。

from lAEVAL import app

if __name__ == "__main__":
    app.run()

3. 配置uWSGI

nano laeval_uwsgi.ini建立laeval_uwsgi.ini配置文件，使得uWSGI知道应用这些设置。
laeval_uwsgi.ini文件配置内容

[uwsgi]
module = wsgi
master = true
processes = 5
socket = laeval.sock
chmod-socket = 666
vacuum = true
die-on-term = true

callable = app
threads = 2
thunder-lock=True
enable-threads=True

http = 0.0.0.0:9800

chdir=/home/richie/myproject
home= /home/richie/myproject/vevn

[uwsgi]标识让uWSGI知道应用这些配置。并通过module = wsgi指定模块(去除wsgi.py中.py的扩展名)。master = true为主模式启动，processes = 5为产生5个工作进程来处理实际的请求。在生成环境下是使用Nginx处理实际与客户端的连接，让后将请求传递给uWSGI。因为这些组件在同一服务器上运行，首选Unix套接字socket保证安全与速度socket = laeval.sock。并同时修改套接字上的权限chmod-socket = 666，确保读取与写入。vacuum = true则可以在进程停止时清理套接字。die-on-term = true可以对其Upstart init系统和uWSGI系统组件，实现预期行为。callable = app为可调用的应用名，默认为“application”，因此需要设置该参数。并enable-threads=True使能线程。chdir=/home/richie/myproject为App应用目录，home= /home/richie/myproject/vevn为App应用虚拟环境目录。

在Ubuntu命令行中可以使用pwd查看当前路径。

使用uwsgi --ini laeval_uwsgi.ini执行uWSGI Web 服务器，可以在http://192.168.1.7:9800/地址中打开主页。
如果调出“预测图像分数”http://192.168.1.7:9800/imgprediction/模板，将会提示错误ValueError: Buffer dtype mismatch, expected 'SIZE_t' but got 'int'。这是因为在机器学习部分，在本地通过pickle.dump()保存的模型clf.pkl、erf.pkl、feartK.pkl和feature_map.pkl，在服务器端无法读取该格式。因此需要直接在服务器端移除这个4个.pkl文件，再执行python imgPred_buildFeatures.py,python imgPred_training.py,重新生成.pkl文件。

4. 建立Upstart Script

Upstart脚本将允许Ubuntu的初始化init系统在服务器引导时自动启动uWSGI,并未flask应用程序提供服务。在/etc/init目录下建立.conf结尾的脚本文件.

sudo nano /etc/init/myproject.conf

myproject.conf文件内容：

description "uWSGI server instance configured to serve myproject"

start on runlevel [2345]
stop on runlevel [!2345]

setuid user
setgid www-data

env PATH=/home/richie/myproject/venv/bin
chdir /home/richie/myproject
exec uwsgi --ini laeval_uwsgi.ini

start on runlevel [2345]定义系统启动脚本的条件。正常的系统运行时间数是2,3,4和5,因此告知系统在达到其中一个运行级时启动脚本。而stop on runlevel [!2345]告知在其它运行级别时停止。例如服务器的重新启动，关闭或者单用户模式时。setuid user和setgid www-data定义运行uWSGI的用户和组。Nginx服务器在www-data组下运行，因此给定读写权限。为正确查找到项目文件，需要设置PATH虚拟环境路径和chdir项目目录，并调用uWSGI可执行文件传入到前文配置的laeval_uwsgi.ini配置文件。
执行下述代码，启动uWSGI。

sudo start myproject

5. 配置Nginx

uWSGI应用程序服务器启动运行后，等待项目目录中套接字文件的请求。需要配置Nginx，使用uwsgi协议将Web请求传递给套接字。
在/etc/nginx/sites-available/下建立配置文件myproject，文件名一般和App项目名同。

sudo nano /etc/nginx/sites-available/laeval

myproject文件内容为：

server {
    listen 80;
    server_name 192.168.1.7;

    location / {
        include uwsgi_params;
        uwsgi_pass unix:/home/richie/myproject/laeval.sock;
    }
}

listen 80;设置Nginx监听默认端口。server_name 192.168.1.7;设置请求服务器的端口或服务器域名。location /事件为一个匹配每个请求的位置块。其中包含include uwsgi_params;uwsgi_params文件指定了一些需要设置的通用uWSGI参数。然后将请求传递给使用 uwsgi_pass指令定义的套接字。
启用创建的Nginx服务器配置文件，将其链接到站点目录。

sudo ln -s /etc/nginx/sites-available/laeval /etc/nginx/sites-enabled

可以通过sudo nginx -t命令测试配置文件是否存在语法错误。

richie@richie-nuc:~/myproject$ sudo nginx -t
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful

如果返回提示successful，则重启Nginx进程读取新配置。

sudo service nginx restart

Nginx启动、停止与重启命令

sudo service nginx start
sudo service nginx stop
sudo service nginx restart

通过sudo service nginx restar启动Nginx，通过uwsgi --ini laeval_uwsgi.ini启动uWSGI，则可以直接在WIFI局域网中在http://192.168.1.7/地址打开网站。

6. supervisor进程守护程序

部署完之后，虽然网站已经正常运行的，但是总会有意外情况发生，因此可以使用supervisor守护进程，在程序崩溃时，可以重新启动。
首先安装supervisor程序

sudo apt-get install supervisor

在/etc/supervisor文件夹下，可以看到supervisord.conf配置文件,和conf.d文件夹，因为supervisord.conf文件下包含[include]files = /etc/supervisor/conf.d/*.conf ;语句，引入/etc/supervisor/conf.d/ 下的 .conf 文件，因此可以在conf.d文件夹下增加配置文件 。

cd /etc/supervisor
root@richie-nuc:/etc/supervisor# ls
conf.d  supervisord.conf

在conf.d文件夹下sudo nano myproject.conf建立myproject.conf文件，在增加下述代码。

[program:myproject]          
command = uwsgi --ini /home/richie/myproject/laeval_uwsgi.ini 
directory = /home/richie/myproject
user = www-data
stopsignal=QUIT
autostart=true
autorestart=true
stdout_logfile=/var/log/uwsgi/supervisor_flask.log     
stderr_logfile=/var/log/uwsgi/supervisor_flask_err.log

启动supervisord

sudo supervisorctl reload
